---
title: SetManager
description: Transfer the pool manager authority to a new account.
---

## Overview

| Property | Value |
|----------|-------|
| Instruction Index | 10 |
| Required Authority | Manager |

The `SetManager` instruction transfers the pool's manager authority to a new account. The new manager must sign to accept the role.

## Accounts

| Index | Name | Writable | Signer | Description |
|-------|------|----------|--------|-------------|
| 0 | Stake Pool | ✅ | ❌ | Stake pool account |
| 1 | Manager | ❌ | ✅ | Current pool manager |
| 2 | New Manager | ❌ | ✅ | New manager (must sign to accept) |
| 3 | New Fee Account | ❌ | ❌ | New manager's fee token account |

## Arguments

None.

## Behavior

1. Validates current manager signature
2. Validates new manager signature (acceptance)
3. Validates new fee account is valid token account for pool mint
4. Updates `manager` to new manager's pubkey
5. Updates `manager_fee_account` to new fee account

## Examples

### Rust

```rust
use spl_stake_pool::instruction::set_manager;

fn transfer_manager(
    client: &RpcClient,
    stake_pool: &Pubkey,
    current_manager: &Keypair,
    new_manager: &Keypair,
    new_fee_account: &Pubkey,
) -> Result<(), Box<dyn std::error::Error>> {
    let set_manager_ix = set_manager(
        &spl_stake_pool::id(),
        stake_pool,
        &current_manager.pubkey(),
        &new_manager.pubkey(),
        new_fee_account,
    );
    
    let tx = Transaction::new_signed_with_payer(
        &[set_manager_ix],
        Some(&current_manager.pubkey()),
        &[current_manager, new_manager], // Both must sign
        client.get_latest_blockhash()?,
    );
    
    client.send_and_confirm_transaction(&tx)?;
    
    println!("Manager transferred to {}", new_manager.pubkey());
    Ok(())
}
```

### CLI

```bash
# Transfer manager role
spl-stake-pool set-manager <POOL_ADDRESS> \
    --new-manager <NEW_MANAGER_KEYPAIR> \
    --new-fee-receiver <NEW_FEE_TOKEN_ACCOUNT>
```

## Errors

| Error | Cause |
|-------|-------|
| `SignatureMissing` | Manager or new manager didn't sign |
| `WrongManager` | Signer is not current manager |
| `WrongAccountMint` | Fee account has wrong mint |
| `InvalidFeeAccount` | Fee account is invalid |

## Security Considerations

- **Both Signatures Required**: Prevents accidental or malicious transfers
- **Atomic Transfer**: Manager and fee account update together
- **Immediate Effect**: New manager has full control after transaction
